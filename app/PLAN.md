Основание: [PROJECT.md](./PROJECT.md)

Цель: по URL англоязычной статьи выполнить парсинг и через AI сгенерировать нужный ответ для каждой кнопки.

---

## Общий поток (для всех трёх кнопок)

```
URL → парсинг (date, title, content) → промпт для AI → OpenRouter → результат в UI
```

---

## Порядок действий

### Шаг 1. Вынести общую подготовку текста статьи ✅

**Файл:** `lib/article-text.ts`

- Функция `buildArticlePromptInput(title, content, maxLength?)` — формирует текст для отправки в AI.
- Обрезка длинного контента (как в `translateArticle`).
- Использовать во всех AI-функциях, чтобы не дублировать код.

---

### Шаг 2. Расширить `lib/openrouter.ts` — три AI-функции ✅

| Функция | Кнопка | Ожидаемый результат |
|---------|--------|---------------------|
| `summarizeArticle(title, content)` | «О чем статья?» | Краткое описание на русском (2–4 абзаца) |
| `extractTheses(title, content)` | «Тезисы» | Список ключевых тезисов на русском |
| `generateTelegramPost(title, content, date?)` | «Пост для Telegram» | Готовый пост для канала на русском |

Модель: `deepseek/deepseek-chat`.

---

### Шаг 3. API-маршруты ✅

`POST /api/analyze` — `{ "url": "...", "action": "summary" | "theses" | "telegram" }`

Ответ: `{ "result": "..." }`.

---

### Шаг 4. Подключить фронтенд ✅

**Файл:** `app/components/ReferentForm.tsx`

- Вызов `/api/analyze` для трёх кнопок.
- Тексты загрузки и вывод текста AI в блоке «Результат».

---

### Шаг 5. Обработка ошибок и краевых случаев ✅

| Случай | Решение |
|--------|---------|
| Статья не распарсилась | Сообщение «Не удалось извлечь текст статьи» |
| Сайт блокирует запрос (403) | Подсказка попробовать другой URL |
| Слишком длинная статья | Обрезка в `buildArticlePromptInput` |
| Нет `OPENROUTER_API_KEY` | Понятная ошибка с указанием `.env.local` / Vercel |
| Таймаут | 15 с — загрузка страницы, 90 с — AI, 120 с — клиент |

**Файлы:** `lib/errors.ts`, `lib/api-helpers.ts`

---
